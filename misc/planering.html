1. Idébeskrivning

Jag tänker göra sidan som jag designade i Gränssnitts kursen vilket var en bok publicerings sida/bok läsnings sida. Sidorna kommer visa; alla böcker som lagts upp, en sammanfattning av en bok man valt, en sida där man läser kapitlet/boken, en sida med inställningar samt en sök/mer avancerad filtringsfunktions sida för genrer. Min målgrupp är primärt blivande författare/folk som skriver för att det är kul, samt sekundärt inriktat till intresserade läsare. Färgtemat kommer vara mörkare grå/svarta färger med ljusare text samt en grön accent färg.

2. Planering

2.1 Handskiss

Mobil skisser: Skisserna till hem, sammanfattningen och läs-sidan på första kollumnen i den ordningen + grid till höger om varje. Till höger om alla skisser gjorde jag en skiss om hur gridet skulle se ut och vilka element som låg i. Pilarna på utsidan ska representera scroll, och dom på insidan margin. Kanske är viktigt att notera, men alla måtten med fractions ändrades på flesta sidorna eftersom mina ögonmått var fel. Skisserna till sidomenyn, sök och profil-sidan i första kolumnen i den ordningen. Samma som den första ritade jag ett grid till höger om varje sida för att förtydliga hur och vilka element skulle vara med. Desktop skisser: Första skissen till hur index hemsidan skulle ungefär se ut. Måtten på hur stora griden var fick ändras. Jag ville också ha mer borders och liknande runt omkring många av elementen i alla sidor på dator versionen. Dock ville inte bilder och borders samarbeta alls. Nav-baren längst är 11vh och finns på alla sidor, den byter ut alla tillbaka knappar/pilar som fanns på mobilsidan. Det finns också en sökbar, den är tänkt att byta ut hela söksidan som också fanns på mobilen. Här liknade dator versionen mobil verionen rätt så mycket. En skillnade jag ville få ihop var hur kapitlerna låg. Men en sån funktion hann jag inte få ihop och det fick bli som mobil versionen istället. Samma som innnan fick grid storlekarna ändras från skissen. Här lade jag till tre aside delar som kan användas till mycket saker. Min tanke låg på att det kan vara som reklam platser eller länkar till andra böcker. Annars liknar den mobil sidan rätt så mycket. Denna sidan var tänkt vara ett alteranativ till sidomenyn som fanns på mobilsidan. Jag tänkte använda mig av elementen som jag redan hade skapat här. Många av dessa sidor har margin till höger och vänster, detta eftersom jag tycker det blir snyggare. Här ser man också att det ligger grid i grid, en main och i den en aside och två sections. Profilen var tänkt ha samma horizontal scroll som mobilen med en extra kanppt till att kunna se alla element (som inte funkar btw).

5. Utvärdering

För det första var planerningen till projektet inte bra. Det var eftesom jag inte visste redan innan hur lång tid kodningen skulle ta och ledde till att mycket blev kvar till sista veckan. Detta projekt var baserat på mitt gränssnitts projekt hösten innan och gjorde det lätt att skapa mobil skisserna då jag redan hade en bas att ta ifrån. Mitt val att gå mobil-first var en bra ide då jag kunde snabbare få ihop dektop versioner genom att återanvända elemten som redan fanns. Det skulle nog ha hjälp att faktiskt ha dekstop skisserna redo från början istället, som jag gjorde att börja på det efter mobil versionen var klar. Min process skulle både förbättrats och effektiviserats om jag hade alla skisser klara vid första veckan. Detta påverkade också projektet och speciellt designs skillnaden mellan mobil och desktop. Under projektets gång hade jag självklart lärt mig mycket om HTML och CSS så när det var dags att skapa desktop skisser skiljde sig designen åt rätt mycket jämnfört med mobilversionen. Planering följdes någonlunda men jag överskattade mig själv med hur mycket jag skulle arbeta hemma. En annan sak som kunde förbättras var planeringen kring hur jag använde classes. Det blev ofta liknande element som jag skrev helt nytt css som istället kunde tagit en klass som redan fanns. Men planeringen som krävs innan kodningen börjar för att riktigt effektivisera class användning och undvika onödig kod måste vara väldigt väl uttänkt, något som jag inte visste om eller hade gjort. Inför nästa gång fokuserar jag på planeringen, inte bara för layout, men också för class användningen. Optimeringen var inte min starka sida och är en sak jag får arbeta med i framtiden för att förbätra. I detta projekt hadde jag dock inte tillräckligt med tid till att förkorta css dokumentet, även om det fanns förbättringar att göras. Jag tror att endast 1000 linjer max skulle ha räckt emot de 1500+ som jag har nu. Ett sätt att förbättra som jag kommer på är att lägga typsnittet längst upp i CSS för alla html dokument "html { font-family: sergou;}" istället för det jag gjorde med att skriva det på varenda element som hade text. Jag hade också mycket strul med min horizontal-scroll på profil sidan. Jag försökte få en border runt den för att inte smälta in i backgrunden så mycket, men med sättet jag kodade in scrollen så hamnade bordern utanför sidan. Den fungerar fortfarande inte perfekt, och jag skulle vilja försöka få ihop en scroll som faktiskt fungerera någon gång. Samtidigt så märker om man jämnfört med indexen jag skapade först för två månader sen och den sista sidan jag gjode klart idag, hur bättra jag har blivit på kodning. Mycket av mistagen jag gjorde då har jag lärt mig av och undviker nu, min process har effektiviserats. Men det blir fortfarande mycket fel som tar tid att felsöka. Om jag tittar tillbaka på processen och kodningen, spenderade jag ca 80% av tiden på felsökning. Den tiden kunde undvikts om mina skisser hade varit mer omfattande tror jag. En sak som jag märkte var hur enheten på hur stort saker var ibland inte ville vara helt responsiva. Enheten som användes mest var rem samt vh och vw. Även om dessa enheter är responsiva så tror jag att vid vissa tillfällen hade % varit bättre. Men också px, för element som ska ha samma längd oberoende storleken på skärmen. Just nu märker man när man ändrar storleken på skärmen hur vissa element hamnar fel. Jag har verkligen försökt få allt att se snyggt ut men vissa saker gick bara inte att fixa där jag Inte hittade lösningar. Jag besultade mig för att göra en till sjätte sida till mobilen i slutet. Det var för att enligt Emil räknades inte en sidomeny som en sida. Även om han godkännde min skiss första veckan där jag hade 5 sidor där sidomenyn var inräknad bara för att vara helt säker. Sammanfattningsvis fick jag spendera mycket tid den sista veckan på att få klart allt vilket inte var så bra. För att undvika en sådan situation i framtiden vill jag fokusera på att planera bättre och sätta hårdare deadlines på mig själv. Om jag hade en deadline på tex två sidor varje vecka hade kanske inte detta varit lika stressigt att få ihop. Några konkreta förbättringsprocesser som jag tänker på kan vara bättre optimisation, alltså undvika onödig kod. Med att uttnyttja klasser mer, men också att använda mig mer av "parent/children" med tex fonter. En väldigt konkret förbättring med min CSS skulle att ha lagt bara en 'font-family:' i toppen av dokumentet så alla texter fick den eftersom jag ändå bara använde en font totalt.
