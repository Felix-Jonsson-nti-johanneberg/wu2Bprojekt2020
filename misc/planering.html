5. Utvärdering

För det första var planerningen till projektet inte bra. Det var eftesom jag inte visste redan innan hur lång tid kodningen skulle ta och ledde till att mycket blev kvar till sista veckan. Mitt val att gå mobil-first var en bra ide då jag kunde snabbare få ihop dektop versioner genom att återanvända elemten som redan fanns. Det skulle nog ha hjälp att faktiskt ha dekstop skisserna redo från början istället, som jag gjorde att börja på det efter mobil versionen var klar. Min process skulle både förbättrats och effektiviserats om jag hade alla skisser klara vid första veckan. Detta påverkade också projektet och speciellt designs skillnaden mellan mobil och desktop. Under projektets gång hade jag självklart lärt mig mycket om HTML och CSS så när det var dags att skapa desktop skisser skiljde sig designen åt rätt mycket jämnfört med mobilversionen. En annan sak som kunde förbättras var planeringen kring hur jag använde classes. Det blev ofta liknande element som jag skrev helt nytt css som istället kunde tagit en klass som redan fanns. Men planeringen som krävs innan kodningen börjar för att riktigt effektivisera class användning och undvika onödig kod Optimeringen var inte min starka sida och är en sak jag får arbeta med i framtiden för att förbätra. I detta projekt hadde jag dock inte tillräckligt med tid till att förkorta css dokumentet, även om det fanns förbättringar att göras. Jag tror att endast 1000 linjer max skulle ha räckt emot de 1500+ som jag har nu. Ett sätt att förbättra som jag kommer på är att lägga typsnittet längst upp i CSS för alla html dokument "html { font-family: sergou;}" istället för det jag gjorde med att skriva det på varenda element som hade text. Samtidigt så märker om man jämnför med indexen jag skapade först för två månader sen och den sista sidan jag gjode klart idag, hur bättra jag har blivit på kodning. Mycket av mistagen jag gjorde då har jag lärt mig av och undviker nu, min process har effektiviserats. Men det blir fortfarande mycket fel som tar tid att felsöka. Om jag tittar tillbaka på processen och kodningen, spenderade jag ca 80% av tiden på felsökning.
